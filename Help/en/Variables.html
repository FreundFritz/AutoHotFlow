Variables

Variable formats

There are some defined variable formats:
Normal
The value can be a number or some text. If a mathematical operation is applied to a string containing a number, it will be automatically interpreted as a number
Date
The value contains a date. If the date is shown, e.g. in a message box, it will be converted to a readable form. For example: 4:55 PM Saturday, November 27, 2004
List
The value contains a list with some elements. Each element has keys. Those keys can either be numerical (comparable to an array) or they can be an arbitrary name.
You can add or access elements by using some specific actions. If the date is shown, e.g. in a message box, it will be converted to a readable form.

Variable names

Variable names are not case sensitive (for example, CurrentDate is the same as currentdate). Variable names may be up to 253 characters long and may consist of letters, numbers and the following punctuation: # _ @ $

A variable name musn't start with the letters "A_", since those are built in variables.

Global and local variables

Local variables

As local variable can have almost any name. As soon it is set, it is available in the current instance. An instance is one execution. As soon the instance ends, all local variables will be deleted.

Global variables

Global variables must begin with "Global_". For example: Global_MyVar. Those variables are available in all flows and kept when the flow has ended, even after a reboot.

Using variables

Variables can be used in almost every edit field in the settings. If a string is needed, a variable's content can be inserted by enclosing them into percent signs.
Example:

The content of Var is: %Var%

Expression

Expressions are used to perform one or more operations upon a series of variables, literal strings, and/or literal numbers. In other words, they may be used in order to do some calculations or to make some comparisons.

Variable names in an expression are not enclosed in percent signs. Numbers are allowed.

It is possible to make a double reference. If a variable is enclosed in percent signs within an expression (e.g. %Var%), whatever that variable contains is assumed to be the name or partial name of another variable.

Examples:

VarContainingFive + 4 * VarContainingThree (results to 17)

(1 or 0) and 1 (results to 1)

number > 5 && number <= 7 (results to 1 if number is between 6 or 7)

Var%VarContainingTwo% (Accessing variable Var2)

Following operands are supported:

+

Add

-

Subtract or Unary minus. This operator may be set in front of a number to make it negative.

*

Multiply

/

Divide.

=

Equal. Case insensetive. This operator is a comparison. It does not assign a value to a variable.

==

Equal. Case sensetive.

!= or <>

Not equal.

>

Greater

<

Less

>=

Greater or Equal

<=

Smaller or Equal

! or NOT

Logical-NOT If the operand is blank or 0, the result of applying logical-not is 1, which means "true". Otherwise, the result is 0 (false).

|| or OR

Logical-OR

&& or AND

Logical-AND

Built in variables

The following variables are built in and are read only.

Special characters

A_Space

This variable contains a single space character.

A_Tab

This variable contains a single tab character.

A_Linefeed or A_LF

This variable contains a single linefeed.

Flow variables

A_TriggerTime

This variable contains the execution time in date format.

Some elements, especially loops and triggers may provide other read only variables. See in the description of each element.
In contrast to other variables, the variables that are set by the actions are thread variables. So, if you separate the flow into two threads, the thread 1 will not access the variables of thread 2.

Date and time

A_Now

The current local time in date format.

A_NowUTC

The current Coordinated Universal Time (UTC) in date format.

A_YYYY or A_Year

Current 4-digit year (e.g. 2004).

A_MM or A_Mon

Current 2-digit month (01-12).

A_DD or A_MDay

Current 2-digit day of the month (01-31).

A_MMMM

Current month's full name in the current user's language, e.g. July.

A_MMM

Current month's abbreviation in the current user's language, e.g. Jul.

A_DDDD

Current day of the week's full name in the current user's language, e.g. Sunday.

A_DDD

Current day of the week's 3-letter abbreviation in the current user's language, e.g. Sun.

A_WDay

Current 1-digit day of the week (1-7). 1 is Sunday in all locales.

A_YDay

Current day of the year (1-366).

A_YWeek

Current week of the year (1-53).

A_Hour

Current 2-digit hour (00-23).

A_Min

Current 2-digit minute (00-59).

A_Sec

Current 2-digit second (00-59).

A_MSec

Current 3-digit millisecond (000-999).

A_TickCount

The number of milliseconds since the computer was rebooted. The precision is 10ms.

A_TimeIdle

The number of milliseconds that have elapsed since the system last received keyboard, mouse, or other input. This is useful for determining whether the user is away. Physical input from the user as well as artificial input generated by any program or script (such as the actions "Send keystrokes" and "Click") will reset this value back to zero.

A_TimeIdlePhysical

Similar to above but ignores artificial keystrokes and/or mouse clicks.

Operating system and user info

A_OSVersion

One of the following strings, if appropriate:
WIN_7, WIN_8, WIN_8.1, WIN_VISTA, WIN_2003, WIN_XP, WIN_2000.

A_Is64bitOS

Contains 1 (true) if the OS is 64-bit or 0 (false) if it is 32-bit.

A_Language

The system's default language, which is a 4-digit code.

A_LanguageName

Name of the system's default language.

A_ComputerName

The name of the computer as seen on the network.

A_UserName

The logon name of the user who launched this flow.

A_IsAdmin

If the flow has admin rights, it contains 1. Otherwise it contains 0.

A_ScreenWidth and A_ScreenHeight

The width and height of the primary monitor, in pixels (e.g. 1024 and 768).

A_ScreenDPI

Number of pixels per logical inch along the screen width. In a system with multiple display monitors, this value is the same for all monitors. On most systems this is 96; it depends on the system's text size (DPI) setting.

A_IPAddress1 through A_IPAddress4

The IP addresses of the first 4 network adapters in the computer.

Directories

A_WorkingDir

The full path and name of the working directory. This the default directory that is used to access files and folders when an absolute path has not been specified. The working directory can be set in the flow settings.

A_Desktop

The full path and name of the folder containing the current user's desktop files.

A_DesktopCommon

The full path and name of the folder containing the all-users desktop files.

A_MyDocuments

The full path and name of the current user's "My Documents" folder.

A_StartMenu

The full path and name of the current user's Start Menu folder.

A_StartMenuCommon

The full path and name of the all-users Start Menu folder.

A_Programs

The full path and name of the Programs folder in the current user's Start Menu.

A_ProgramsCommon

The full path and name of the Programs folder in the all-users Start Menu.

A_Startup

The full path and name of the Startup folder in the current user's Start Menu.

A_StartupCommon

The full path and name of the Startup folder in the all-users Start Menu.

A_ProgramFiles

The Program Files directory (e.g. C:\Program Files).

A_WinDir

The Windows directory. For example: C:\Windows

A_AppData

The full path and name of the folder containing the current user's application-specific data. For example: C:\Documents and Settings\Username\Application Data.

A_AppDataCommon

The full path and name of the folder containing the all-users application-specific data.

A_ScriptDir

The full path of the directory where the current AutoHotFlow is located. The final backslash is omitted (even for root directories).

A_ScriptFullPath

The full path of the executable of the flow editor.
