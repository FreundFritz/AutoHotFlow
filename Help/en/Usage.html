Usage



Quick Start

Start Base Frame. Press on Open and choose your main uncompiled script.
Fill all edit fields on the left-hand side.


Application name:


Name of the application.


Abbreviation:


The abbreviation of the application name.


Long name:


Name of the application which may be longer.


Installation folder:


Folder where the installation files will be unpacked.


Application version:


Short application version number whitch will be shown during install.


Update Version:


Precise application version number. It will be used for the comparison of the current version and the online available version.


Update URL:


Link to the .ini file which contains the Informations about the current online available version.


Select files on right-hand side, whitch will be packed into the installer.


Folders:


All subfolders in the installed application.


Files:


All files that will be packet into the installation file and extracted during installation.


Exe:


All scripts, whitch will be compiled and packed into the installation file.


Optionally you can choose an icon for each of your scripts that should be compiled.


Press the button "Generate installation file".

Your settings will be saved in setting.ini. You wont need set everything again next time.



Details

Application name, Abbreviation and long name will be shown during the Installation.

They don't have any other importance.

The update function will be started when the main application will start. It checks in the background whether an update is available. The automatic Update check can be disabled:

  • In the file settings.ini you need to add following entry:

    [BaseFrame]
    Check for updates=0


    A 0 inactivates the update function. I recommend to add a setting into your application whitch lets the user to decide whether to check for updates automatically changes this entry.

The update function will work when you consider following:

  • There must be a link to the update comparison file. The link must lead directly to the file.

  • The comparison file must be formatted like an ini file. Example:

    [Update Info]
    Version=2.00.00
    Download Path=https://dl.dropbox.com/s/...
    Open Site=http://de.autohotkey.com/forum/...

  • The comparison file must contain whitch information is available and either the direct download link to the installer or a webpage. Or both.

  • The comparison version numbers will be compared as following:

     if (DieseVersion < DownloadedVersion) 

    You must make sure, that the version number of any new application is greater than the other.
    I recommend following format: 2.02.14

What does Base Frame

  • As first step it generates the sourcecodes for all three fuctions:

    • Update Checker.ahk

    • "Application name" modified.ahk

    • "Application name" Installer.ahk


    The original quellcode files of the appications will be left untouched.

  • The three scripts (and eventually other that you have set) will be compiled.

  • The installation file will be compiled.
